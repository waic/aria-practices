アコーディオンの例

この例について

以下の例のセクションには、アコーディオンパターンのデモンストレーションとして、3 つのセクションに分けられたシンプルな個人情報入力フォームが含まれています。

例

キーボードのサポート

キー 機能
Space 又は Enter 折りたたまれたセクションのアコーディオンヘッダーにフォーカスがある場合、そのセクションを展開します。
Tab
  • 次のフォーカス可能な要素にフォーカスを移動します。
  • アコーディオン内のすべてのフォーカス可能な要素は、ページの Tab シーケンスに含まれます。
Shift + Tab
  • 前のフォーカス可能な要素にフォーカスを移動します。
  • アコーディオン内のすべてのフォーカス可能な要素は、ページの Tab シーケンスに含まれます。

ロール、プロパティ、ステート、及び tabindex 属性

ロール 属性 要素 使用法
h3
  • アコーディオンヘッダーとして機能する要素。
  • 各アコーディオンヘッダー要素には、コンテンツパネルの可視性を制御するボタンが含まれています。
  • この例ではページのアウトラインに正しく適合するために、見出しレベル 3 が使用されています。この例は、レベル 2 の見出しでタイトル付けられたセクションの中に含まれているためです。
aria-expanded="true" button アコーディオンパネルが展開されている場合は true に設定され、それ以外の場合は false に設定されます。
aria-controls="ID" button ヘッダーが制御するパネルの ID を指します。
region div 現在展開されているアコーディオンパネルを含むランドマークの領域を生成します。
aria-labelledby="IDREF" div
  • region 要素のアクセシブルな名前を定義します。
  • 領域を展開する及び折りたたむアコーディオンヘッダーボタンを参照します。
  • region 要素は、ランドマークとして識別されるためにアクセシブルな名前を持つ必要があります。
