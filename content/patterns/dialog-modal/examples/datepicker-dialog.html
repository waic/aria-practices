日付選択ダイアログの例

この例の概要

以下の例は、日付入力欄と、ダイアログ (モーダル) パターンを実装した日付選択欄を開くボタンで構成されています。 ダイアログには、利用者がカレンダーから日付を選択できるようにするボタンを配置するためにグリッドパターンを使用したカレンダーが含まれています。 カレンダーから日付を選択すると、ダイアログが閉じられ、日付入力欄に選択された日付が入力されます。 ダイアログが開かれたとき、入力欄が空であるか、有効な日付が入力されていない場合、カレンダー内の現在日がフォーカスされます。 そうでない場合、日付入力欄の値と一致するカレンダー内の日付にフォーカスが配置されます。

類似例:

  • アラートダイアログの例: アラートダイアログをデモンストレーションする確認プロンプトです。
  • モーダルダイアログの例: 少量と大量のコンテンツ両方を持つ複数のモーダルダイアログのレイヤーをデモンストレーションする例です。
  • 日付選択コンボボックス: カレンダーグリッドと、月と年を移動するためのボタンを含むダイアログを開く、編集可能な日付入力コンボボックスです。

アクセシビリティ機能

  • 日付形式の説明は、aria-describedby を介してテキスト入力と関連付けられているため、支援技術でアクセシブルな説明として利用できます。
  • 日付が選択されると、「日付を選択」ボタンのアクセシブル名は「日付を変更、{日付文字列}」に変更されます。ここで、日付文字列は選択された日付です。 そのため、ダイアログが閉じられ、フォーカスがボタンに戻ったとき、スクリーンリーダー利用者は選択された日付の確認を聞きます。
  • ダイアログでは、カレンダーに表示される月と年を変更するための追加ボタンにショートカットキーが割り当てられています。
  • 月と年を表示するカレンダーの見出しはライブリージョンとしてマークアップされているため、スクリーンリーダー利用者は、月と年を変更するボタンやキーボードコマンドからのフィードバックを受け取ります。
  • キーボードヘルプはダイアログの下部に表示されます。 フォーカスがカレンダーグリッドに移動したとき、スクリーンリーダー利用者に通知するためにライブリージョンが使われます。
  • カレンダーでのコンパクトな視覚デザインを容易にするために、列ヘッダーの曜日名は 2 文字に省略されています。 ただし、これにより、スクリーンリーダー 利用者にとって曜日名を理解することが難しくなります。 そのため、列ヘッダーの HTML abbr 属性で支援技術に完全な曜日名を提供することで、利用者がグリッドを移動するときにスクリーンリーダーが完全な名前を読み上げられるようになります。 (訳註: 原文では、 Monday を Mo と表記するような英語の曜日名の省略について解説していますが、日本語の曜日名は 1 文字で表記されるため、省略の問題は軽微でしょう。)
  • ダイアログボックスのコントロールのフォーカスとホバーのスタイリングに対する高コントラストサポートは、CSS の border プロパティを使用します:
    • ボタン又は日付セルがフォーカスを受けたとき、ボーダーが追加されます。
    • ポインティングデバイスでボタン又は日付セルの上にホバーしたとき、ボーダーが追加されます。
    • デフォルトでは、ボタンと日付セルにはボーダーがなく、パディングがフォーカスとホバーのスタイリングのために追加されたボーダーのプレースホルダーとして使用されます。

キーボードサポート

日付を選択ボタン

キー 機能
Space、
Enter
  • 日付選択ダイアログを開きます。
  • 選択された日付、つまり日付入力テキスト欄に表示されている日付にフォーカスを移動します。 日付が選択されていない場合は、現在の日付にフォーカスを置きます。

日付選択ダイアログ

キー 機能
ESC ダイアログを閉じ、「日付を選択」ボタンにフォーカスを戻します。
Tab
  • ダイアログのTab シーケンス内の次の要素にフォーカスを移動します。
  • グリッドパターンで定められているように、カレンダーグリッド内のボタンは 1 つのみが Tab シーケンス内にあります。
  • 最後のボタン (つまり、「OK」) にフォーカスがある場合、最初のボタン (つまり、「前の年」) にフォーカスを移動します。
Shift + Tab
  • ダイアログのTab シーケンス内の前の要素にフォーカスを移動します。
  • グリッドパターンで定められているように、カレンダー グリッド内のボタンは 1 つのみが Tab シーケンス内にあります。
  • 最初のボタン (つまり、「前の年」) にフォーカスがある場合、最後のボタン (つまり、「OK」) にフォーカスを移動します。

Page Down
  • 日付のグリッドを次の月に変更します。

Shift + Page Down
  • 日付のグリッドを翌年の同じ月に変更します。
  • 同じ日付の日にフォーカスを移動します。その日がない場合は、その月の最後の日にフォーカスを移動します。

日付選択ダイアログ: OK 及びキャンセルボタン

キー 機能
Space、
Enter		 ボタンを作動します：
  • 「キャンセル」: ダイアログを閉じ、「日付を選択」ボタンにフォーカスを移動し、日付入力欄の日付を更新しません。
  • 「OK」: ダイアログを閉じ、「日付を選択」ボタンにフォーカスを移動し、日付入力欄の日付を更新し、「日付を選択」ボタンのアクセシブル名を選択した日付を含むように更新します。

日付を選択ボタン

ロール 属性 要素 使用法
aria-label="String" button
  • アクセシブル名の初期値は「日付を選択」です。
  • 利用者が日付を選択したとき、アクセシブル名は「日付を変更、{日付文字列}」に変更されます。ここで、日付文字列は選択された日付です。

日付選択ダイアログ

ロール 属性 要素 使用法
dialog div ダイアログとして要素を識別します。
aria-modal="true" div ダイアログがモーダルであることを示します。
aria-label="string" div ダイアログのアクセシブル名を定義します。
aria-live="polite" h2
  • 月及び／又は年の変更したとき、h2 要素の内容が更新されます。
  • スクリーンリーダーによって自動的に読み上げられることを示します。
aria-live="polite" div
  • グリッドを移動するためのキーボードコマンドに関する情報を表示する要素が、スクリーンリーダーによって自動的に読み上げられるべきであることを示します。
  • スクリプトによって情報の表示がわずかに遅延されるため、スクリーンリーダーはフォーカスの変更に関連する情報を読み上げた後に、この情報をより読みやすくなります。

日付選択ダイアログ: 日付グリッド

ロール 属性 要素 使用法
grid table
  • table 要素を grid ウィジェットとして識別します。
  • grid ロールが table 要素に適用されるため、row、columnheader、gridcell ロールは、tr、th、td タグによって暗黙的に指定されるため、指定する必要はありません。
aria-labelledby="ID_REFERENCE" table grid のアクセシブル名を指定する要素を識別します。
tabindex="0" td
  • セルをフォーカス可能にし、ダイアログの Tab シーケンスに含めます。
  • 要素がダイアログの Tab シーケンスに含まれるときに、JavaScript によって動的に設定されます。
  • 一度に、グリッド内の 1 つの gridcell のみがダイアログの Tab シーケンス内にあります。
  • フォーカスの管理に関するこのアプローチは、「Roving tabindex を使用したコンポーネント内のフォーカスの管理」セクションで説明されています。
tabindex="-1" td
  • セルをフォーカス可能にし、ダイアログの Tab シーケンスから除外します。
  • セルがダイアログの Tab シーケンスに含まれるときに、JavaScript によって動的に 0 に変更されます。
  • 一度に、グリッド内の 1 つの gridcell のみがダイアログの Tab シーケンス内にあります。
  • フォーカスの管理に関するこのアプローチは、「Roving tabindex を使用したコンポーネント内のフォーカスの管理」セクションで説明されています。
aria-selected="true" td
  • 現在選択されている日付、つまり日付入力欄に存在する日付値を含むセルを識別します。
  • 現在選択されているセルにのみ設定され、他のセルには aria-selected は指定されません。

注: 列ヘッダーの曜日の名前は 2 文字に省略されているため、スクリーンリーダーで読み上げられると理解しにくくなる可能性があります。 th 要素に abbr 属性を適用することで、スクリーンリーダーに代替の列ヘッダー名を提供できます。 そのため、各 th 要素には、その列の曜日の完全なスペルを含む abbr 属性が含まれています。 (訳註: 原文では、 Monday を Mo と表記するような英語の曜日名の省略について解説していますが、日本語の曜日名は 1 文字で表記されるため、省略の問題は軽微でしょう。)
