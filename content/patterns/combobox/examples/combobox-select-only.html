セレクトオンリーコンボボックスの例

この例について

次の例は、Combobox Pattern のシングルセレクトコンボボックスウィジェットの実装例を示しています。これは HTML の select 要素と機能的に似ています。 編集可能なコンボボックスの例とは異なり、このセレクトオンリーコンボボックスは <input> 要素で作られておらず、自由形式の利用者入力を受け付けません。 しかし、HTML の <select> と同様に、利用者は文字を入力して一致するオプションを選択できます。

類似の例には以下が含まれます:

  • リストとインラインオートコンプリートを備えた編集可能なコンボボックス: リストとインラインオートコンプリート として知られるオートコンプリート動作を示す編集可能なコンボボックス。
  • リストオートコンプリートを備えた編集可能なコンボボックス: リストと手動選択 として知られるオートコンプリート動作を示す編集可能なコンボボックス。
  • オートコンプリートなしの編集可能なコンボボックス: aria-autocomplete=none に関連する動作を示す編集可能なコンボボックス。
  • グリッドポップアップを備えた編集可能なコンボボックス: 提案をグリッドで表示し、各提案に関する説明情報をナビゲートできる編集可能なコンボボックス。
  • 日付選択コンボボックス: カレンダーグリッドと月及び年をナビゲートするためのボタンを含むダイアログを開く編集可能な日付入力コンボボックス。

アクセシビリティ機能

この例の機能と利用者エクスペリエンスは、属性 size="1" を持つ HTML の select 要素とほぼ同等ですが、アクセシビリティと一般的な使いやすさを向上させるために、以下の動作の違いが実装されています。

  1. コンボボックスが折りたたまれている場合、利用者が印刷可能な文字を入力すると、リストボックスが表示され、aria-activedescendant を介してアクセシビリティフォーカスを受け取ります。 これにより、利用者はオプションの存在を認識し、支援技術利用者はオプションのリストのサイズを理解できます。
  2. オプションのリストをナビゲートしても、入力の値は設定されません。 これにより、オプションを認識するためにナビゲートする必要があるスクリーンリーダー利用者は、入力の現在の値を失うことなくオプションを探索できます。 利用者が スペースキー、Enter、又は Tab を押すか、フォーカスがコンボボックスから外れると、値が設定されます。 Escape キーでリストボックスを閉じるか、入力をクリックしてリストを折りたたむと、現在の値が保持されます。
  3. ブラウザは、フォーカスのある要素と同様に aria-activedescendant によって参照される要素の可視性を管理しません。 キーボードイベントがリストボックス内のアクティブなオプションを変更すると、JavaScript は aria-activedescendant によって参照されるオプションを表示領域にスクロールします。 aria-activedescendant の可視性を管理することは、コンテンツのサイズを大きくするためにブラウザのズーム機能を使用する人々にとってアクセシビリティのために不可欠です。

キー 機能
下矢印
  • リストボックスが表示されていない場合にリストボックスを開きますが、フォーカスや選択を変更しません。
  • DOM フォーカスはコンボボックスに残ります。
Alt + 下矢印 フォーカスや選択を変更せずにリストボックスを開きます。
上矢印

  • DOM フォーカスはコンボボックスに残ります。
Enter フォーカスや選択を変更せずにリストボックスを開きます。

印刷可能な文字
  • リストボックスが表示されていない場合にまずリストボックスを開き、次に入力された文字に一致する最初のオプションに視覚的フォーカスを移動します。
  • 複数のキーが短時間で連続して入力された場合、視覚的フォーカスは入力された文字列全体に一致する最初のオプションに移動します。
  • 同じ文字が連続して入力された場合、視覚的フォーカスはその文字で始まるオプションの間を循環します。

リストボックスポップアップ

注記: 視覚的フォーカスがリストボックスにあるとき、DOM フォーカスはコンボボックスに残り、コンボボックスの aria-activedescendant の値は視覚的にフォーカスが示されているリストボックスオプションを参照する値に設定されます。 以下のキーボードコマンドの説明でフォーカスに言及している場合、それは視覚的フォーカスインジケーターを指しています。 このフォーカス管理テクニックの詳細については、 aria-activedescendant を使用したコンポジット内のフォーカス管理 を参照してください。

Tab
  • リストボックス内のフォーカスされたオプションの内容に値を設定します。
  • リストボックスを閉じます。
  • デフォルトの動作を実行し、フォーカスを次のフォーカス可能な要素に移動します。 注記: ネイティブの <select> 要素はリストボックスを閉じますが、タブではフォーカスを移動しません。 このパターンは、この場合、ネイティブ要素ではなく他のコンボボックスの動作に一致します。
Escape
  • リストボックスを閉じます。
  • 視覚的フォーカスをコンボボックスに設定します。
下矢印キー

  • 視覚的フォーカスが最後のオプションにある場合、視覚的フォーカスは移動しません。
上矢印キー
  • 視覚的フォーカスを前のオプションに移動します。
  • 視覚的フォーカスが最初のオプションにある場合、視覚的フォーカスは移動しません。
Alt + 上矢印キー
  • リストボックス内のフォーカスされたオプションの内容に値を設定します。
  • リストボックスを閉じます。
  • 視覚的フォーカスをコンボボックスに設定します。
Home 視覚的フォーカスを最初のオプションに移動します。
End 視覚的フォーカスを最後のオプションに移動します。
PageUp 視覚的フォーカスを 10 オプション上にジャンプします (又は最初のオプションに)。
PageDown 視覚的フォーカスを 10 オプション下にジャンプします (又は最後のオプションに)。
印刷可能な文字

ロール、プロパティ、ステート、及び tabindex 属性

このページの例のコンボボックスは、以下の ARIA ロール、ステート、及びプロパティを実装しています。 他の ARIA ロール、ステート、及びプロパティの適用方法については、コンボボックスパターンのロール、ステート、及びプロパティのセクションをご覧ください。

コンボボックス

ロール 属性 要素 使用法
combobox div 入力をコンボボックスとして特定します。
aria-labelledby="#IDREF" div コンボボックスをラベル付けする要素を特定します。
aria-controls="#IDREF" div ポップアップとして機能する要素を特定します。
aria-expanded="false" div ポップアップ要素が表示されていないことを示します。
aria-expanded="true" div ポップアップ要素が表示されていることを示します。
aria-activedescendant="IDREF" div
  • リストボックス内のオプションがキーボードフォーカスを持つことを視覚的に示すとき、そのオプションを参照します。
  • 下矢印キーなどのナビゲーションキーが押されたとき、JavaScript が値を変更します。
  • 支援技術が、DOM フォーカスが input 要素に残っている間にアプリケーションがフォーカスしている要素を知ることができます。
  • このフォーカスマネジメントテクニックの詳細については、 aria-activedescendant を使用したコンポジットのフォーカス管理をご覧ください。

ロール 属性 要素 使用法
listbox div 要素をlistboxとして特定します。
option div

  • 要素のテキストコンテンツがoptionのアクセシブルネームを提供します。
