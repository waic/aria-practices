日付ピッカー コンボボックスの例

この例について

以下の日付ピッカーは、ダイアログを開く コンボボックスパターン の実装を示しています。 日付ピッカーダイアログは、日付を選択するためのダイアログを開くために、日付を選択するボタンをアクティブ化するか、キーボードフォーカスをコンボボックスに移動して Down Arrow または Alt + Down Arrow を押すことで開かれます。 ダイアログには、カレンダーを表示し、日付の選択を可能にするための グリッドパターン の実装が含まれています。 ダイアログ内の追加のボタンを使用して、グリッドに表示される月と年を変更することもできます。

類似の例には以下があります:

  • 日付ピッカーダイアログの例: 日付を選択するためのカレンダーグリッドを含むダイアログを示しています。
  • 単一選択コンボボックス: テキスト入力のない単一選択コンボボックスで、HTMLの select 要素と機能的に似ています。
  • リストとインラインオートコンプリートを備えた編集可能なコンボボックス: インラインオートコンプリートとして知られるオートコンプリートの動作を示す編集可能なコンボボックスです。
  • リストオートコンプリートを備えた編集可能なコンボボックス: マニュアル選択として知られるオートコンプリートの動作を示す編集可能なコンボボックスです。
  • オートコンプリートのない編集可能なコンボボックス: aria-autocomplete=none に関連する動作を示す編集可能なコンボボックスです。
  • グリッドポップアップを備えた編集可能なコンボボックス: 各提案に関する説明情報をナビゲートできるグリッドで提案を表示する編集可能なコンボボックスです。

アクセシビリティ機能

  • 日付形式の説明は、コンボボックスと aria-describedby を使用して関連付けられ、アシスト技術にアクセス可能な説明として提供されます。
  • ダウンアローのアイコンは、combobox デザインパターンで指定されているように Tab シーケンスから除外されますが、アシスト技術には「日付を選択」ボタンとしてアクセス可能になっています。 これにより、キーボードを持たないアシスト技術のユーザー（タッチベースのスクリーンリーダーを使用しているユーザーなど）が日付ピッカーダイアログを開くことができます。
  • ダイアログでは、月と年を変更するための追加のボタンにショートカットキーが割り当てられています。
  • キーボードヘルプはダイアログの下部に表示されます。フォーカスがグリッドに移動すると、ライブリージョンがスクリーンリーダーユーザーにアナウンスされます。
  • 月と年を変更するボタンやキーボードコマンドからのフィードバックをスクリーンリーダーユーザーが受け取るため、月と年を表示するカレンダーの見出しはライブリージョンとしてマークアップされています。
  • カレンダーのコンパクトなビジュアルデザインをサポートするため、列ヘッダの曜日名は2文字に省略されています。ただし、これによりスクリーンリーダーユーザーが曜日名を理解するのが困難になります。そのため、列ヘッダのHTMLの abbr 属性に完全な曜日名が提供され、スクリーンリーダーがグリッドをナビゲートする際に完全な名前をアナウンスできるようになっています。
  • コントロールのフォーカスとホバースタイルは、CSSのボーダープロパティを使用してオペレーティングシステムの高コントラスト設定をサポートしています：
    • ボタンや日付セルがフォーカスを受け取ると、ボーダーが追加されます。
    • ポインティングデバイスでボタンや日付セルにホバーすると、ボーダーが追加されます。
    • デフォルトでは、ボタンや日付セルにはボーダーがありません。フォーカスとホバースタイルのプレースホルダースペースとしてパディングが提供されます。

ロール、プロパティ、ステート、および tabindex 属性

コンボボックス

ロール 属性 要素 使用法
combobox input input 要素をコンボボックスとして特定します。
aria-haspopup="dialog" input コンボボックスがダイアログを開くことを示します。
aria-expanded="false" input コンボボックスが折りたたまれていること、つまり「日付を選択」ダイアログが表示されていないことを示します。

aria-autocomplete="none" input コンボボックスがオートコンプリートをサポートしていないことを示します。
aria-controls="IDREF" input コンボボックスによって制御される要素を特定します。
aria-describedby="IDREF" input コンボボックスのアクセシブルな説明を提供する要素を特定し、支援技術が日付形式の説明を input と関連付けることを可能にします。

日付ピッカーダイアログ: 日付グリッド

ロール 属性 要素 使用法
grid table
  • table 要素を grid ウィジェットとして識別します。
  • grid ロールが table 要素に適用されているため、row、columnheader、および gridcell ロールは tr、th、および td タグによって暗黙的に指定されるため、明示的に指定する必要はありません。
aria-labelledby="IDREF" table グリッドのアクセシブルな名前を提供する要素を識別します。これは、グリッドに表示される月と年を示す h2 です。
tabindex="0" td
  • グリッドセルをフォーカス可能にし、ダイアログの Tab シーケンスに含めます。
  • 要素がダイアログの Tab シーケンスに含まれるときに JavaScript によって動的に設定されます。
  • 任意の時点で、グリッド内の 1 つのグリッドセルのみがダイアログの Tab シーケンスに含まれます。
  • このフォーカス管理のアプローチは、ロービング tabindex を使用したコンポーネント内のフォーカス管理 のセクションで説明されています。
tabindex="-1" td
  • グリッドセルをフォーカス可能にし、ダイアログの Tab シーケンスから除外します。
  • グリッドセルがダイアログの Tab シーケンスに含まれるときに JavaScript によって動的に 0 に変更されます。
  • 任意の時点で、グリッド内の 1 つのグリッドセルのみがダイアログの Tab シーケンスに含まれます。
  • このフォーカス管理のアプローチは、ロービング tabindex を使用したコンポーネント内のフォーカス管理 のセクションで説明されています。
aria-selected="true" td
  • セルが選択されていることを示します。
  • コンボボックスの現在の値を表すグリッドセルに設定されます。他のグリッドセルには aria-selected が指定されていません。

注記: 列ヘッダーの日の名前が 2 文字に省略されているため、スクリーンリーダーで読み上げられると理解しにくい場合があります。 列ヘッダーの名前の代替をスクリーンリーダーに提供するために、th 要素に abbr 属性を適用することができます。 したがって、各 th 要素には、その列の日の名前の完全なスペルを含む abbr 属性が含まれています。
