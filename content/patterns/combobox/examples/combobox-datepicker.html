日付選択コンボボックスの例

この例について

以下の日付選択は、ダイアログを開くコンボボックスパターンの実装を示しています。 日付選択ダイアログは、日付を選択するボタンをアクティブ化するか、キーボードフォーカスをコンボボックスに移動して Down Arrow 又は Alt + Down Arrow を押すことで開かれます。 ダイアログには、カレンダーを表示し、日付の選択を可能にするための グリッドパターン の実装が含まれています。 グリッドに表示される月と年を変更するために、ダイアログ内の追加のボタンを使用できます。

類似の例には以下があります:

  • 日付選択ダイアログの例: 日付を選択するためのカレンダーグリッドを含むダイアログを示しています。
  • 選択限定コンボボックス: テキスト入力のない単一選択コンボボックスで、HTML の select 要素と機能的に似ています。
  • リストとインラインオートコンプリートの両方を備えた編集可能コンボボックス: 「リストとインラインのオートコンプリート」として知られるオートコンプリートの動作を示す編集可能なコンボボックスです。
  • リストオートコンプリート付き編集可能コンボボックス: オートコンプリートの動作である「リストと手動選択」として知られるオートコンプリートを示す編集可能なコンボボックスです。
  • オートコンプリートなし編集可能コンボボックス: aria-autocomplete=none に関連する動作を示す編集可能なコンボボックスです。
  • グリッドポップアップ付き編集可能コンボボックス: グリッドで提案を表示し、各提案の説明情報をナビゲートできる編集可能なコンボボックスです。

アクセシビリティ機能

  • 日付形式の説明は、コンボボックスと aria-describedby を使用して関連付けられ、支援技術にアクセス可能な説明として提供されます。
  • 下矢印のアイコンは、combobox デザインパターンで指定されているように Tab シーケンスから除外されていますが、支援技術には「日付を選択」ボタンとしてアクセス可能になっています。 これにより、キーボードを持たない支援技術の利用者 (タッチベースのスクリーンリーダーを使用している利用者など) が日付選択ダイアログを開くことができます。
  • ダイアログでは、カレンダーに表示されている月と年を変更するための追加のボタンにショートカットキーが割り当てられています。
  • キーボードヘルプはダイアログの下部に表示されます。フォーカスがグリッドに移動すると、ライブリージョンがスクリーンリーダー利用者にアナウンスされます。
  • 月と年を表示するカレンダーの見出しはライブリージョンとしてマークアップされているため、スクリーンリーダー利用者はボタン及びキーボードコマンドが月及び年を変更するフィードバックを受け取ります。
  • カレンダーのコンパクトなビジュアルデザインを容易にするため、列ヘッダの曜日名は 2 文字に省略されています。 ただし、これによりスクリーンリーダー利用者が曜日名を理解するのが困難になります。 そのため、列ヘッダの HTML の abbr 属性に完全な曜日名が提供され、スクリーンリーダーがグリッドをナビゲートする際に完全な名前をアナウンスできるようになっています。 (訳註: 原文は英語であり、"Sunday" を "Su" と表すようなケースについて言及しています。)
  • コントロールのフォーカス及びホバースタイルは、CSS のボーダープロパティを使用してオペレーティングシステムの高コントラスト設定をサポートしています:
    • ボタンや日付セルがフォーカスを受け取ったとき、ボーダーが追加されます。
    • ポインティングデバイスでボタンや日付セルにホバーしたとき、ボーダーが追加されます。
    • デフォルトでは、ボタン及び日付セルにはボーダーがありません。フォーカスとホバースタイルのプレースホルダースペースとしてパディングが提供されます。

キーボードのサポート

コンボボックス

キー 機能
Down Arrow,
ALT + Down Arrow
  • 日付選択ダイアログを開きます。
  • コンボボックスに有効な日付が含まれている場合、カレンダーグリッド内のその日付にフォーカスを移動します。 それ以外の場合は、現在の日付 (つまり、今日の日付) にフォーカスを移動します。

日付選択ダイアログ

キー 機能
ESC ダイアログを閉じ、フォーカスをコンボボックスに移動します。
TAB
  • ダイアログの Tab シーケンス内の次の要素にフォーカスを移動します
  • ただし、グリッドパターン で指定されているように、カレンダーグリッド内の要素は 1 つだけが Tab シーケンスに含まれます。
  • フォーカスが最後のボタン (つまり、「OK」) にある場合、最初のボタン (つまり、「前の年」) にフォーカスを移動します。
Shift + TAB
  • ダイアログの Tab シーケンス内の前の要素にフォーカスを移動します。
  • ただし、グリッドパターン で指定されているように、カレンダーグリッド内の要素は 1 つだけが Tab シーケンスに含まれます。
  • フォーカスが最初のボタン (つまり、「前の年」) にある場合、最後のボタン (つまり、「OK」) にフォーカスを移動します。

日付選択ダイアログ: OK とキャンセルボタン

キー 機能
Space,
Enter 		ボタンをアクティブ化します:
  • 「キャンセル」: ダイアログを閉じ、コンボボックスにフォーカスを移動し、コンボボックスの値を更新しません。
  • 「OK」: ダイアログを閉じ、コンボボックスにフォーカスを移動し、コンボボックスの日付を更新します。

ロール、プロパティ、ステート、及び tabindex 属性

コンボボックス

ロール 属性 要素 使用法
combobox input input 要素をコンボボックスとして識別します。
aria-haspopup="dialog" input コンボボックスがダイアログを開くことを示します。
aria-expanded="false" input コンボボックスが折りたたまれていること、つまり「日付を選択」ダイアログが表示されていないことを示します。
aria-expanded="true" input コンボボックスが展開されていること、つまり「日付を選択」ダイアログが開いていることを示します。
aria-autocomplete="none" input コンボボックスがオートコンプリートをサポートしていないことを示します。
aria-controls="IDREF" input コンボボックスによって制御される要素を識別します。
aria-describedby="IDREF" input コンボボックスのアクセシブルな説明を提供する要素を特定し、支援技術が日付形式の説明を入力欄と関連付けることを可能にします。

日付選択ボタン

ロール 属性 要素 使用法
tabindex="-1" button コンボボックスのデザインパターンで指定されているように、ボタン (つまり、下矢印アイコン) を Tab シーケンスから除外します。
aria-label="string" button アクセシブルな名前を「日付を選択」と定義し、ボタンを作動することで開かれるダイアログのタイトルと一致させます。

日付選択ダイアログ

ロール 属性 要素 使用法
dialog div 要素をダイアログとして識別します。
aria-modal="true" div ダイアログがモーダルであることを示します。
aria-label="日付を選択" div ダイアログのアクセシブルな名前を定義します。
aria-live="polite" h2
  • 現在表示されている月と年が変更されたときに、スクリーンリーダーが自動的に要素をアナウンスすることを示します。
  • polite 値は、他のアナウンスが中断されるべきでないことを示します。
aria-live="polite" div
  • グリッドのナビゲーションに関するキーボードコマンドの情報を表示する要素が、スクリーンリーダーによって自動的にアナウンスされるべきであることを示します。
  • スクリプトは情報の表示を少し遅らせるため、スクリーンリーダーがフォーカス変更イベントのアナウンスの後にアナウンスする可能性が高くなります。
  • polite 値は、他のアナウンスが中断されるべきでないことを示します。

日付選択ダイアログ: 日付グリッド

ロール 属性 要素 使用法
grid table
  • table 要素を grid ウィジェットとして識別します。
  • grid ロールが table 要素に適用されているため、row、columnheader、及び gridcell ロールは tr、th、及び td タグによって暗黙的に指定されるため、明示的に指定する必要はありません。
aria-labelledby="IDREF" table grid のアクセシブルな名前を提供する要素を識別します。これは、グリッドに表示される月と年を示す h2 です。
tabindex="0" td
  • グリッドセルをフォーカス可能にし、ダイアログの Tab シーケンスに含めます。
  • 要素がダイアログの Tab シーケンスに含まれるときに JavaScript によって動的に設定されます。
  • 任意の時点で、グリッド内の 1 つのグリッドセルのみがダイアログの Tab シーケンスに含まれます。
  • このフォーカス管理のアプローチは、動き回る tabindex を使用したコンポーネント内のフォーカス管理 のセクションで説明されています。
tabindex="-1" td
  • グリッドセルをフォーカス可能にし、ダイアログの Tab シーケンスから除外します。
  • グリッドセルがダイアログの Tab シーケンスに含まれるときに JavaScript によって動的に 0 に変更されます。
  • 任意の時点で、グリッド内の 1 つのグリッドセルのみがダイアログの Tab シーケンスに含まれます。
  • このフォーカス管理のアプローチは、動き回る tabindex を使用したコンポーネント内のフォーカス管理 のセクションで説明されています。
aria-selected="true" td
  • セルが選択されていることを示します。
  • コンボボックスの現在の値を表すグリッドセルに設定されます。他のグリッドセルには aria-selected が指定されていません。

注記: 列ヘッダの曜日の名前が 2 文字に省略されているため、スクリーンリーダーで読み上げられると理解しにくい場合があります。 列ヘッダの名前の代替をスクリーンリーダーに提供するために、th 要素に abbr 属性を適用することができます。 したがって、各 th 要素には、その列の日の名前の完全なスペルを含む abbr 属性が含まれています。 (訳註: 原文は英語であり、"Sunday" を "Su" と表すようなケースについて言及しています。)
