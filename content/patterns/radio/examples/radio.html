「tabindex の移動 (roving tabindex)」を使用したラジオグループの例

この例について

この例では、ラジオグループ パターンの機能を、二つのラジオグループのために実装しています -- ひとつはピザのクラストを選ぶためで、もうひとつは受け渡しの方法を選ぶためです。この実装では、ラジオグループ内でフォーカスを管理するために「tabindex の移動 (roving tabindex)」を使用しています。

類似の例には以下があります:

  • aria-activedescendant を使用したラジオグループの例: キーボードフォーカスを管理するために aria-activedescendant を使用するラジオボタングループ。
  • 評価用のラジオグループの例: 五つ星の評価スケールに入力を提供するラジオグループ。

アクセシビリティ機能

  • aria-checked ステートと視覚的な状態表示を同期させるために、CSS の属性セレクタを使用します。
  • 視覚的なキーボードフォーカス及びホバーのスタイリングに、CSS の :hover 及び :focus 疑似クラスを使用します。
    • フォーカスインジケータはラジオボタン及びラベルの両方を囲むため、どのオプションが選ばれているかを、認識しやすくなっています。
    • ホバーはラジオボタン及びラベルの両方の背景を変更するため、ラベルあるいはボタンのいずれをクリックしてもラジオボタンが作動することを、認識しやすくなっています。
    • ラジオボタンの上にホバーすると、カーソルの形状がポインタに変わるため、視覚障害のある人々がそれをインタラクティブな要素であると識別するのに役立ちます。
  • 透明なボーダーは、ハイコントラストが有効になっているシステムで表示されてしまうことがあるため、フォーカスされた要素にのみ可視のボーダーを持たせています。フォーカス可能な要素がフォーカスされていない場合は、その要素に対して幅が 0 のボーダー及び、フォーカスを示すのに使われるボーダーと同じ幅の追加的なパディングを持たせています。
  • ハイコントラストの設定によってテキストコンテンツの色が変更されたときに、CSS ファイルの中にあるインライン SVG のラジオボタンのグラフィックが背景と十分なコントラストを持つように、svg のグラフィックにおいて、CSS の forced-color-adjust は auto に設定されています。これにより、circle 要素の stroke 及び fill の色は、ハイコントラストの色設定によって上書きされます。circle 要素の背景をハイコントラスト時の背景色と一致させるために、外側の circle の fill-opacity 属性はゼロに設定され、内側の circle の stroke-opacity 属性もゼロに設定されています。forced-color-adjust が stroke 及び fill 属性で指定された色を上書きするのに用いられない場合、それらの色はハイコントラストモードでも変わらないままとなってしまい、ラジオボタンの circle 要素と背景との間のコントラストが不十分になるか、あるいはそれらの色がハイコントラストモードの背景色と一致する場合、見えなくなる可能性すらあります。
    注記: forced-color-adjust の明示的な設定は必要です。というのも、一部のブラウザでは SVG のグラフィックにおいて、デフォルト値として auto が用いられないからです。

キーボードのサポート

キー 機能
Tab
  • radiogroup 内の選択された radio ボタンにフォーカスを移動する。
  • radio ボタンが選択されていない場合、グループ内の最初の radio ボタンにフォーカスを移動する。
スペース
  • フォーカスが当たっている radio ボタンが選択されていない場合、ステートを checked に変更する。
  • それ以外の場合、何もしない。
  • 注記: ラジオボタンが選択されていない状態は、ページ読み込み時にのみ発生する。
下矢印
右矢印
  • グループ内の次の radio ボタンにフォーカスを移動し、選択する。
  • フォーカスが最後の radio ボタンにある場合、最初の radio ボタンにフォーカスを移動する。
  • 以前に選択されていたラジオボタンのステートは、選択されていない状態に変更される。
上矢印
左矢印
  • グループ内の前の radio ボタンにフォーカスを移動し、選択する。
  • フォーカスが最初の radio ボタンにある場合、最後の radio ボタンにフォーカスを移動し、選択する。
  • 以前に選択されていたラジオボタンのステートは、選択されていない状態に変更される。

ロール、プロパティ、ステート、及び tabindex 属性

ロール 属性 要素 使用法
radiogroup div
  • div 要素を radio ボタンのグループのコンテナとして明示する。
  • 以下で説明するように、フォーカスは「tabindex の移動 (roving tabindex)」戦略を使用して管理されるため、フォーカス可能ではない。
aria-labelledby="[IDREF]" div ラジオグループのラベルを含む要素を参照する。
radio div
  • div 要素を ARIA の radio ボタンとして明示する。
  • アクセシブルな名前は、div 要素の子テキストコンテンツから割り出される。
tabindex="-1" div
  • 要素をフォーカス可能にするが、ページの Tab シーケンスの一部ではない。
  • ページの Tab シーケンスに含まれる一つのラジオボタンを除く、ラジオグループ内のすべてのラジオボタンに適用される。
  • このフォーカス管理のアプローチは、 「tabindex の移動 (roving tabindex)」を使用したコンポーネント内のフォーカス管理というプラクティスで説明されている。
tabindex="0" div
  • ラジオボタンをフォーカス可能にし、ページの Tab シーケンスに含める。
  • ラジオグループ内で、一つのラジオボタンにのみ設定される。
  • ページの読み込み時に、ラジオグループ内の最初のラジオボタンに設定される。
  • ラジオグループ内でフォーカスが移動すると、最近にフォーカスされたラジオボタンがページの Tab シーケンスに含まれるようになる。
  • このフォーカス管理のアプローチは、 「tabindex の移動 (roving tabindex)」を使用したコンポーネント内のフォーカス管理というプラクティスで説明されている。
aria-checked="false" div
  • 選択されていない radio ボタンを特定。
  • CSS の属性セレクタ (例: [aria-checked="false"]) を使用して、視覚的な状態と aria-checked 属性の値を同期させる。
  • オペレーティングシステム及びブラウザのハイコントラスト設定をサポートすべく、CSS の ::before 疑似要素が、選択されていないラジオボタンの視覚的な状態を示すために使用される。
aria-checked="true" div
  • 選択されている radio ボタンを特定。
  • CSS の属性セレクタ (例: [aria-checked="true"]) を使用して、視覚的な状態と aria-checked 属性の値を同期させる。
  • オペレーティングシステム及びブラウザのハイコントラスト設定をサポートすべく、CSS の ::before 疑似要素が、選択されているラジオボタンの視覚的な状態を示すために使用される。
