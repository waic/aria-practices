Roving tabindexを使用したラジオグループの例

この例について

この例は、2つのラジオグループのためのラジオグループパターンの機能を実装しています -- 1つはピザのクラストを選ぶためで、もう1つは配送方法を選ぶためです。 この実装では、ラジオグループ内でフォーカスを管理するためにロービング tabindex を使用しています。

類似の例には以下のものがあります:

  • aria-activedescendant を使用したラジオグループの例: キーボードフォーカスを管理するために aria-activedescendant を使用するラジオボタングループ。
  • 評価ラジオグループの例: 5つ星の評価スケールに入力を提供するラジオグループ。

アクセシビリティ 機能

  • aria-checked ステートと視覚ステートインジケータを同期させるために CSS 属性セレクタを使用。
  • CSS の :hover 及び :focus 疑似クラスを使用して、視覚キーボードフォーカス及びホバーのスタイリング。
    • フォーカスインジケータはラジオボタン及びラベルの両方を含むため、どのオプションが選ばれているかを認識しやすくする。
    • ホバーはラジオボタン及びラベルの背景を変更し、ラベル又はボタンのいずれかをクリックするとラジオボタンが作動することを認識しやすくする。
    • ラジオボタンの上にホバーすると、カーソルがポインタに変わり、視覚障害のある人々がそれをインタラクティブ要素として特定するのを助ける。
  • 透明な境界線は、ハイコントラストが有効ないくつかのシステムで可視であるため、フォーカス可能な要素がフォーカスされていないとき、フォーカスを示すために使用される境界線と同じ幅の境界線と追加のパディングを持つ0幅の境界線があります。
  • CSS ファイルのインライン SVG ラジオボタングラフィックが、ハイコントラスト設定がテキストコンテンツの色を変更するときに背景と十分なコントラストを持つように、CSS の forced-color-adjust は svg グラフィックの auto に設定されています。 これにより、stroke 及び fill のcircle 要素の色がハイコントラスト色設定によって上書きされます。 circle 要素の背景をハイコントラスト背景色と一致させるために、外側の circle の fill-opacity 属性はゼロに設定され、内側の circle の stroke-opacity 属性はゼロに設定されています。 forced-color-adjust が stroke 及び fill プロパティで指定された色を上書きするために使用されなかった場合、それらの色はハイコントラストモードで同じままであり、ラジオサークル要素と背景の間に不十分なコントラストをもたらすか、それらの色がハイコントラストモードの背景と一致する場合、それらを見えなくする可能性があります。
    注記: forced-color-adjust の明示的な設定は必要であり、一部のブラウザは SVG グラフィックのデフォルト値として auto を使用しないためです。

キーボードサポート

キー 機能
Tab
  • radiogroup内の選択されたradioボタンにフォーカスを移動する。
  • radioボタンが選択されていない場合、グループ内の最初のradioボタンにフォーカスを移動する。
Space
  • フォーカスが当たっているradioボタンが選択されていない場合、checkedのステートに変更する。
  • それ以外の場合、何もしない。
  • 注記: ラジオが選択されていない状態は、ページ読み込み時にのみ発生します。
下矢印
右矢印
  • グループ内の次のradioボタンにフォーカスを移動し、選択する。
  • フォーカスが最後のradioボタンにある場合、最初のradioボタンにフォーカスを移動する。
  • 以前に選択されたラジオボタンのステートは選択されていないに変更される。
上矢印
左矢印
  • グループ内の前のradioボタンにフォーカスを移動し、選択する。
  • フォーカスが最初のradioボタンにある場合、最後のradioボタンにフォーカスを移動し、選択する。
  • 以前に選択されたラジオボタンのステートは選択されていないに変更される。

ロール、プロパティ、ステート、及び Tabindex 属性

ロール 属性 要素 使用法
radiogroup div
  • div 要素をradioボタンのグループのコンテナとして特定する。
  • 以下で説明するように、フォーカスはロービング tabindex 戦略を使用して管理されるため、フォーカス可能ではありません。
aria-labelledby="[IDREF]" div ラジオグループのラベルを含む要素を参照する。
radio div
  • div 要素を ARIA radio ボタンとして特定する。
  • アクセス可能な名前は、div 要素の子テキストコンテンツから計算される。
tabindex="-1" div
  • 要素をフォーカス可能にし、ページの Tab シーケンスの一部には含まない。
  • ラジオグループに含まれるすべてのラジオボタンに適用され、ページの Tab シーケンスに含まれるものは1つだけである。
  • このフォーカス管理のアプローチは、 ロービング tabindex を使用したコンポーネント内のフォーカス管理 のプラクティスで説明されています。
tabindex="0" div
  • ラジオボタンをフォーカス可能にし、ページの Tab シーケンスに含める。
  • ラジオグループ内のラジオに1つだけ設定される。
  • ページの読み込み時に、ラジオグループ内の最初のラジオボタンに設定される。
  • ラジオグループ内でフォーカスが移動すると、最近フォーカスされたラジオボタンがページの Tab シーケンスに含まれるように、フォーカスとともに移動する。
  • このフォーカス管理のアプローチは、 ロービング tabindex を使用したコンポーネント内のフォーカス管理 のプラクティスで説明されています。
aria-checked="false" div
  • 選択されていないradioボタンを特定する。
  • CSS 属性セレクタ (例: [aria-checked="false"]) を使用して、視覚的なステートとaria-checked属性の値を同期させる。
  • CSS ::before疑似要素は、オペレーティングシステム及びブラウザの高コントラスト設定をサポートするために、選択されていないラジオボタンの視覚的なステートを示すために使用される。
aria-checked="true" div
  • 選択されているradioボタンを特定する。
  • CSS 属性セレクタ (例: [aria-checked="true"]) を使用して、視覚的なステートとaria-checked属性の値を同期させる。
  • CSS ::before疑似要素は、オペレーティングシステム及びブラウザの高コントラスト設定をサポートするために、選択されているラジオボタンの視覚的なステートを示すために使用される。
