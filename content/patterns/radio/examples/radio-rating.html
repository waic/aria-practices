評価用のラジオグループの例

この例について

以下は、ラジオグループ パターンのデモンストレーションである、評価入力の例です。評価は、利用者が選択した星の数で示されます。

類似の例には以下があります:

  • 評価用のスライダーの例: 五つ星の評価スケールに対して評価入力の現在値及び最大値を伝えるために aria-valuetext を使用する水平スライダーのデモンストレーション。
  • aria-activedescendant を使用したラジオグループの例: キーボードフォーカスを管理するために aria-activedescendant を使用するラジオボタングループ。
  • 「tabindex の移動 (roving tabindex)」を使用したラジオグループの例: キーボードフォーカスを管理するために「tabindex の移動 (roving tabindex)」を使用するラジオボタングループ。

アクセシビリティの特徴

  • ラジオを操作する際の知覚可能性を向上させるために、視覚的なキーボードフォーカス及びホバーは CSS を使用してスタイルが設定されています:
    • フォーカスリングは幅が 2 ピクセルで、星のボーダーとの間に少なくとも 2 ピクセルの間隔があります。
    • 星の上にホバーするとカーソルがポインタに変わるので、評価用の星がインタラクティブであることがより容易に知覚できます。
  • ハイコントラストの設定で色が反転された際に、星及びフォーカスリングのボーダーが、背景と十分なコントラストを持つことを確実にするために、ボーダーの色はテキストコンテンツの色と同期されています。 例えば、星のボーダーを描くためのインライン SVG の polygon 及びフォーカスリングのボーダーを描くのに使用される rect 要素の stroke 属性に、CSS の currentcolor の値を指定することで、フォーカスリングのボーダーの色は、ハイコントラストモードのテキストの前景色と一致するように設定されます。 polygon及びrect要素の背景をハイコントラストモードの背景色と一致させるために、rect の fill-opacity 属性はゼロに設定されています。もしもそうせずに、stroke 及び fill 属性を設定するのに特定の色が用いられた場合は、それらの色はハイコントラストモードでも同じままであり、星と背景のコントラストが不十分になってしまったり、あるいは、それらの色がハイコントラストモードの背景と一致する場合には、見えなくなってしまう可能性すらあります。
    注記: currentcolor の値がハイコントラストモードで更新されるように、SVG 要素では CSS の forced-color-adjust 属性を auto に設定する必要があります。一部のブラウザはデフォルト値に auto を用いていません。

キーボードのサポート

キー 機能
Tab
  • radiogroup 内の選択された radio ボタンにフォーカスを移動する。
  • radio ボタンが選択されていない場合、グループ内の最初の radio ボタンにフォーカスを移動する。
スペース
  • フォーカスが当たっている radio ボタンが選択されていない場合、ステートを checked に変更する。
  • それ以外の場合、何もしない。
  • 注記: ラジオボタンが選択されていない状態は、ページ読み込み時にのみ発生します。
下矢印
右矢印
  • グループ内の次の radio ボタンにフォーカスを移動し、選択する。
  • フォーカスが最後の radio ボタンにある場合、最初の radio ボタンにフォーカスを移動する。
  • 以前に選択されていたラジオボタンのステートは、選択されていない状態に変更される。
上矢印
左矢印
  • グループ内の前の radio ボタンにフォーカスを移動し、選択する。
  • フォーカスが最初の radio ボタンにある場合、最後の radio ボタンにフォーカスを移動し、選択する。
  • 以前に選択されていたラジオボタンのステートは、選択されていない状態に変更される。

ロール、プロパティ、ステート、及び tabindex 属性

ロール 属性 要素 使用法
none svg SVG 要素上の none ロールの使用は、支援技術が SVG 要素を画像又は他のロールとして解釈しないように保証します。
radiogroup div
  • div 要素を radio ボタンのグループのコンテナとして特定します。
  • 以下で説明するように、フォーカスは「tabindex の移動 (roving tabindex)」戦略を使用して管理されるため、フォーカス可能ではありません。
aria-labelledby="ID_REFERENCE" div ラジオグループのラベルを含む要素を参照します。
radio g g 要素を ARIA の radio ボタンとして特定します。
aria-label="name" g ラジオボタンに関連付けられた星の数を特定するための、アクセシブルな名前を定義します (例: 「一つ星」「二つ星」... )。
tabindex="-1" g
  • 要素をフォーカス可能にしますが、ページの Tab シーケンスの一部ではありません。
  • ページの Tab シーケンスに含まれる一つのラジオボタンを除く、ラジオグループ内のすべてのラジオボタンに適用されます。
  • このフォーカス管理のアプローチは、 「tabindex の移動 (roving tabindex)」を使用したコンポーネント内のフォーカス管理というプラクティスで説明されています。
tabindex="0" g
  • ラジオボタンをフォーカス可能にし、ページの Tab シーケンスに含めます。
  • ラジオグループ内で、一つのラジオボタンにのみ設定されます。
  • ページの読み込み時に、ラジオグループ内の最初のラジオボタンに設定されます。
  • ラジオグループ内でフォーカスが移動すると、最近にフォーカスされたラジオボタンがページの Tab シーケンスに含まれるようになります。
  • このフォーカス管理のアプローチは、 「tabindex の移動 (roving tabindex)」を使用したコンポーネント内のフォーカス管理というプラクティスで説明されています。
aria-checked="false" g
  • 選択されていない radio ボタンを特定します。
  • CSS の属性セレクタ (例: [aria-checked="false"]) を使用して、視覚的な状態と aria-checked 属性の値を同期させます。
  • オペレーティングシステム及びブラウザのハイコントラスト設定をサポートすべく、CSS の ::before 疑似要素が、選択されていないラジオボタンの視覚的な状態を示すために使用されます。
aria-checked="true" g
  • 選択されている radio ボタンを特定します。
  • CSS の属性セレクタ (例: [aria-checked="true"]) を使用して、視覚的な状態と aria-checked 属性の値を同期させます。
  • オペレーティングシステム及びブラウザのハイコントラスト設定をサポートすべく、CSS の ::before 疑似要素が、選択されているラジオボタンの視覚的な状態を示すために使用されます。
