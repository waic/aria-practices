スライド制御のタブを持つ自動回転イメージカルーセルの例

この例について

以下のカルーセルパターンの実装例は、ページの読み込み時に自動的に回転を開始するカルーセルのアクセシビリティに不可欠なパターンの特徴を示しています。 この例は、シーケンス内のスライドをスキップして直接表示するスライドを選択する方法として、タブパターンの使用方法も示しています。 スライド間のナビゲーションは、タブとして機能する一連の円形のボタンによって提供され、回転するスライドの上にレンダリングされます。 ユーザーがカルーセル内の任意のコントロールやリンクにフォーカスを移動するか、カルーセルのコンテンツの上にマウスをホバーすると、自動回転が停止します。 このデザインの詳細を説明するアクセシビリティ機能のセクションが、例の後に続きます。

類似の例には以下があります：

  • スライド制御のボタンを持つ自動回転イメージカルーセル: ページの読み込み時に自動的に回転するカルーセルの必要なアクセシビリティ機能を示す基本的なイメージカルーセルで、前のスライドと次のスライドのボタンで表示するスライドを選択することができます。

例のオプション

カルーセルの表示と動作のオプション

このオプションは、コントロールとキャプションを画像内にレンダリングするよりもアクセシブルであり、コントロールが認識しやすく、回転する画像の背後にないキャプションが読みやすくなります。

このオプションは、ページの読み込み時にカルーセルが一時停止しているか再生中かを制御します。 一時停止オプションは、視覚障害のあるユーザーや自動回転によって気が散ったり混乱したりする人々のアクセシビリティを向上させますが、start/stopボタンを使用して自動回転を開始することができます。

注: ユーザーがシステム設定で動きを減少させるオプションを選択している場合、読み込み時に自動回転は常に一時停止します。

このオプションは、自動回転機能を無効にし、ユーザーインターフェースからstart/stopボタンを削除することで、視覚障害のあるユーザーや自動回転によって気が散ったり混乱したりする人々のアクセシビリティを向上させます。 ユーザーはタブリストを使用してスライドを手動でナビゲートすることができます。

アクセシビリティ機能

自動スライド回転の制御

ユーザーはスライドの回転を停止および開始することができ、これはさまざまな障害を持つ人々のカルーセルのアクセシビリティの重要な側面です。 視覚的な処理や読書に影響を与える認知障害を持つ低視力者や人々は、スライドの内容を探索するのに十分な時間を持つためにスライドの回転を制御できることで利益を得ることができます。 同様に、スクリーンリーダーのユーザーが自動回転を知覚できないため、ページの読み取りが混乱しやすくなります。 例えば、スライドが自動的に回転している場合、スクリーンリーダーのユーザーはスライド1の要素を読み、次の要素を読むためのスクリーンリーダーコマンドを実行するかもしれません。そして、スライド1の次の要素を聞く代わりに、完全に新しいコンテキストからの要素を知らずにスライド2からの要素を聞くことになります。

この例には、ユーザーがスライドの回転を制御するための以下の機能が含まれています:

  • オペレーティングシステムの設定が動きの減少またはアニメーションの無効化に設定されている場合、自動回転は初めて一時停止されます。
  • カルーセルの任意のコンテンツにマウスをホバーすると、自動回転が一時停止します。 キーボードフォーカスなど、回転を防ぐ別の条件がトリガーされていない限り、マウスがカルーセルから移動すると自動回転が再開されます。
  • タブ要素を含むカルーセルの任意のコンテンツにキーボードフォーカスを移動すると、自動回転が一時停止します。 ユーザーが明示的に再生ボタンをアクティブにしない限り、自動回転は再開されません。
  • カルーセルには、自動回転を停止および開始できる回転制御ボタンも含まれています。
    • 回転制御ボタンは、スクリーンリーダーの読み取り順序の最初の要素です。
    • 回転制御ボタンは常に表示されているため、マウス、キーボード、補助技術、またはタッチを介して対話しているすべてのユーザーに利用可能です。
    • カルーセルが回転している場合、ボタンのアクセス可能な名前は自動スライドショーを停止となり、スライドが変更されていることをスクリーンリーダーユーザーに通知するとともに、変更を停止する方法を提供します。
    • カルーセルが回転していない場合、ボタンのアクセス可能な名前は自動スライドショーを開始となります。
    • ユーザーが回転制御ボタンをアクティブにして回転を開始する場合、ユーザーが自動回転をすぐに開始したいと考えていると仮定され、カルーセル内のフォーカスまたはホバー状態は無視されます。回転は、ボタンが再びアクティブにされる場合を除いて、停止することはありません。
  • この例には、自動スライド回転を完全に無効にするオプションが含まれています。 このオプションが選択されると、開始/停止ボタンはレンダリングされず、スライドはタブコントロールのアクティベーションを通じてのみ回転できます。

テキストと回転コントロールのカラーコントラスト

このカルーセルのビューでコントロールとキャプションが画像内に表示されている場合、背景画像によってコントロールとテキストのカラーコントラストが不十分になる可能性があります。 ただし、このビューには、WCAG 2.1のカラーコントラスト要件を満たすための次の機能が含まれています:

  • スライドのレンダリングを制御するタブが画像内にある場合、タブにはWCAG 2.1のカラーコントラスト要件を満たす前景色と背景色があります。 さらに、フォーカススタイリングは、コントロールがキーボードフォーカスを受け取るときにフォーカスインジケーターが非常に目立つようにするSVG画像を使用しています。
  • キャプションエリアの透明度が減少して、キャプションテキストがWCAG 2.1のカラーコントラスト要件を満たすようになります。

画像内のコントロールとテキストのカラーコントラストの問題を回避する方法の1つは、それらを画像の外に移動することです。 コントロールとテキストを固定背景に移動することで、著者がカラーコントラストを制御し、ユーザーがそれらを知覚するのが容易になります。 この技術の例は、ビューオプションの1つです。

TablistとTabsのフォーカススタイリング

  • タブがフォーカスを受け取ると、タブリスト全体もボーダーでハイライトされ、ユーザーにグループ内のコントロール間を移動するために矢印キーが使用されることを示す手がかりが提供されます。
  • フォーカススタイリングに使用されるCSSボーダーの幅は、コントロールがフォーカスを受け取るときに0ピクセルから2ピクセルに変更され、オペレーティングシステムの高コントラストオプションが有効になっているときにフォーカスの可視性をサポートします。

スライド変更のスクリーンリーダーのアナウンスメント

自動回転がオフになっている場合、カルーセルスライドのコンテンツはライブリージョンに含まれています。 これにより、スクリーンリーダーのユーザーがカルーセルのスライドをスキャンするのが容易になります。 スクリーンリーダーのユーザーが新しいタブをアクティブにすると、新しいスライドのコンテンツがアナウンスされ、ユーザーがコンテンツと対話するかどうかを判断するための即時のフィードバックが提供されます。 非常に重要なことに、自動回転がオンになっている場合、ライブリージョンは無効になります。 そうでなければ、継続的に変更されるコンテンツのアナウンスメントがユーザーが読んでいる他のものを常に中断するため、ページは使用できなくなります。

タブパネルをライブにすることは、典型的なタブの場合には推奨されません。 主な目的がコンテンツのクイックで軽量なスキャンであるカルーセルの使用ケースでは、ライブリージョンは、非スクリーンリーダーユーザーが自動回転で持っているタイプの経験をシミュレートするために使用されます。

キーボードサポート

回転制御ボタン

キー 機能
Tab
  • カルーセル内のインタラクティブな要素を通してフォーカスを移動します。
  • 回転制御ボタンとタブリストは、Tabの順序でスライドコンテンツの前に来ます。
Enter または Space カルーセル内のスライドの自動回転を切り替えます。

タブ

キー 機能
Tab
  • フォーカスがtablistに移動すると、アクティブなtab要素にフォーカスを置きます。カルーセル内の各スライドはtabによって制御されます。
  • タブリストがフォーカスを持っている場合、現在表示されているスライドのリンクであるtabの順序で次の要素にフォーカスを移動します。
Right Arrow
  • 次のタブにフォーカスを移動します。
  • フォーカスが最後のタブにある場合、最初のタブにフォーカスを移動します。
  • 新しくフォーカスされたタブに関連付けられたスライドを表示します。
Left Arrow
  • 前のタブにフォーカスを移動します。
  • フォーカスが最初のタブにある場合、最後のタブにフォーカスを移動します。
  • 新しくフォーカスされたタブに関連付けられたスライドを表示します。
Home 最初のタブにフォーカスを移動し、最初のスライドを表示します。
End 最後のタブにフォーカスを移動し、最後のスライドを表示します。

役割、プロパティ、状態、およびTabindex属性

回転制御

役割 属性 要素 使用法
aria-label button アクセス可能な名前は、ボタンが実行するアクションを反映して変更されます：
  • 自動回転が一時停止している場合、ラベルは「自動スライドショーを開始」です。
  • カルーセルが自動的に回転している場合、ラベルは「自動スライドショーを停止」です。

タブ

役割 属性 要素 使用法
暗黙の役割 region section
  • アクセス可能な名前がある任意のsection要素には役割regionが暗黙的に適用されます。
  • カルーセルとそのコントロールをランドマーク領域として定義します。
aria-roledescription="carousel" section
  • 要素を「領域」としてではなく「カルーセル」として識別するように支援技術に通知します。
  • ランドマーク領域への移動のような機能には影響せず、支援技術が役割をどのようにレンダリングするかに影響します。
  • 注: aria-roledescriptionの値はページの言語を使用して定義され、著者によってローカライズされる必要があります。
aria-label="注目のテレビ番組" section カルーセル領域の内容を説明するラベルを提供します。
aria-live="off" div.carousel-items
  • すべてのスライドを含むdiv要素に適用されます。
  • コンテナを「off」状態のライブ領域として識別し、領域の変更が自動的に通知されないことを意味します。
  • カルーセルが自動的に回転している場合、ライブ領域はオフになります。
aria-live="polite" div.carousel-items
  • すべてのスライドを含むdiv要素に適用されます。
  • コンテナ要素を「polite」状態のライブ領域として識別し、次の利用可能な機会に領域の変更が通知されることを意味します。
  • 関連するタブがアクティブになると、スクリーンリーダーが自動的にスライドの内容をアナウンスします。
tablist div 要素がタブのセットのコンテナとして機能することを示します。
aria-label="スライド" div タブリストのアクセス可能な名前を提供します。
tab button
  • 要素がタブ制御として機能することを示します。
  • フォーカスが当たると、自動的にアクティブになり、関連するtabpanel（すなわち、スライド）が表示されます。
aria-label="スライド X" button タブのアクセス可能な名前を提供します。
aria-selected="true" button
  • タブ制御がアクティブであり、関連するパネルが表示されていることを示します。
  • タブがフォーカスを受け取ったときに設定されます。
aria-selected="false" button
  • タブ制御がアクティブでなく、関連するパネルが表示されていないことを示します。
  • 現在表示されているスライドに関連するタブを除く、タブセット内のすべてのタブ要素に設定されます。
tabindex="-1" button
  • 要素をページのTab順序から削除します。
  • 選択されていないタブに設定され、ページのTab順序に選択されたタブのみが含まれるようにします。
  • タブ要素がHTMLのbuttonであるため、タブが選択（アクティブ）されたときにはtabindex属性が削除され、tabindex="0"に設定するのではなく。
  • このフォーカスの管理方法は、 Roving tabindexを使用したコンポーネント内のフォーカスの管理 のセクションで説明されています。
aria-controls="IDREF" button タブに関連するtabpanel要素を参照します。
tabpanel div
  • 要素が、関連するタブ要素によって制御されるtabpanelとして機能することを示します。
  • 関連するtab制御がアクティブでない限り、非表示になります。
aria-roledescription="slide" div
  • 要素を「tabpanel」としてではなく「slide」として識別するように支援技術に通知します。
  • 支援技術の機能には影響せず、支援技術が役割をどのようにレンダリングするかに影響します。
  • 注: aria-roledescriptionの値はページの言語を使用して定義され、著者によってローカライズされる必要があります。
aria-label="X of Y" div
  • スライドのセット内でのスライドの位置を示すタブパネル（スライド）のアクセス可能な名前を提供します。
  • aria-roledescriptionと組み合わせると、スクリーンリーダーはタブパネルをその位置でアナウンスすることができます。例:「スライド1 of 6」。
